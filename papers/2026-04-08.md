## Paper 1: GrandCode: Achieving Grandmaster Level in Competitive Programming via Agentic Reinforcement Learning

- **arXiv:** https://arxiv.org/abs/2604.02721
- **Topics:** Agentic RL, Code Generation, Reinforcement Learning

**Summary:**
GrandCode is a multi-agent reinforcement learning system that has surpassed every human contestant — including top legendary grandmasters — in live Codeforces competitive programming contests. Under standard live contest conditions, GrandCode ranked first in all three Codeforces rounds it entered, finishing all tasks in each contest before any human participant.

The system orchestrates multiple specialized agentic modules (hypothesis proposal, solver, test generator, summarization, etc.) and jointly trains them via a new algorithm called **Agentic GRPO**, designed specifically for multi-stage agent rollouts with delayed rewards and severe off-policy drift. This is the first AI system to definitively beat the world's best competitive programmers in real-time, high-stakes settings.

**Key Contributions:**
- Introduces **Agentic GRPO**, a multi-stage RL algorithm combining immediate reward updates with delayed correction for better credit assignment in long rollouts.
- First AI to achieve grandmaster rank by competing live against humans on Codeforces without special conditions.
- Demonstrates that multi-agent orchestration + online test-time RL can push coding systems beyond the human frontier.

## Paper 2: Scaling Reasoning Tokens via RL and Parallel Thinking: Evidence From Competitive Programming

- **arXiv:** https://arxiv.org/abs/2604.01302
- **Topics:** Reasoning Scaling, Reinforcement Learning, Test-Time Compute

**Summary:**
This paper investigates how reasoning capability scales with the number of generated tokens during RL training and inference. The authors find an approximately **log-linear relationship** between validation accuracy and average reasoning token count across successive RL checkpoints. To exploit this, they introduce a multi-round **parallel thinking pipeline** that distributes a token budget across multiple generation threads and rounds of verify-and-refine, making it practical to scale reasoning without the quadratic cost of single-thread long-context generation.

Starting from Seed-OSS-36B, the full system (16 threads × 16 rounds) matches the model's oracle pass@16 at pass@1 using ~7.6M tokens per problem and surpasses GPT-5-high on 456 hard AetherCode competitive programming problems.

**Key Contributions:**
- Empirical evidence for a **log-linear scaling law** between reasoning tokens and accuracy during RL training.
- **Parallel thinking pipeline** enabling practical scaling of test-time compute without full-attention overhead.
- Verification RL warmup and randomized clipping techniques that shift the scaling trajectory upward.

## Paper 3: RefineRL: Advancing Competitive Programming with Self-Refinement Reinforcement Learning

- **arXiv:** https://arxiv.org/abs/2604.00790
- **Topics:** Reinforcement Learning, Self-Refinement, Code Generation, Efficiency

**Summary:**
RefineRL introduces a **Skeptical-Agent** — an iterative self-refinement agent equipped with local code execution tools — that validates generated solutions against public test cases and enforces rigorous re-examination even when early tests pass. The agent is trained via RL with standard verifiable reward data (no human feedback required). Remarkably, training only on 4.5K problems from the HardTest dataset, a compact 4B-parameter model equipped with RefineRL outperforms the single-attempt performance of 32B models and rivals 235B-parameter models.

**Key Contributions:**
- **Skeptical-Agent** architecture that maintains doubt even on apparently-correct solutions and iterates until confident.
- Demonstrates that **self-refinement is severely under-explored** as a scaling axis; a 4B model trained with RefineRL rivals 235B models.
- Shows that RLVR (RL with verifiable rewards) is sufficient to train effective self-refinement without human annotation.

## Paper 4: VLMs Need Words: Vision Language Models Ignore Visual Detail In Favor of Semantic Anchors

- **arXiv:** https://arxiv.org/abs/2604.02486
- **Topics:** Vision-Language Models, Multimodal AI, Evaluation

**Summary:**
This paper reveals a fundamental limitation of current Vision-Language Models (VLMs): when asked to reason about visual entities, VLMs perform significantly better when those entities are **easily nameable in language** (e.g., "a dog", "a chair") versus entities that are visually distinctive but hard to describe linguistically (faces, abstract shapes, textures). The gap holds across semantic correspondence, shape matching, and face recognition tasks, suggesting VLMs are anchoring on language semantics rather than performing true visual reasoning.

**Key Contributions:**
- Systematic evaluation showing VLMs have a strong **"semantic anchor" bias**: performance degrades substantially for unnameable visual entities.
- Reveals that visual detail is largely ignored by VLMs in favor of language-level semantic categories — a critical blindspot for real-world applications like medical imaging and fine-grained recognition.
- Provides a new diagnostic evaluation protocol to measure the nameable vs. unnameable performance gap in VLMs.

## Paper 5: Selective Forgetting for Large Reasoning Models

- **arXiv:** https://arxiv.org/abs/2604.03571
- **Topics:** Machine Unlearning, Reasoning Models, Safety, Privacy

**Summary:**
Standard machine unlearning techniques suppress a model's final answer about sensitive information, but fail to account for **chain-of-thought (CoT) leakage**: reasoning models can expose the very information targeted for removal through intermediate thinking steps, even when the final answer is suppressed. This paper proposes a targeted solution using a retrieval-augmented multi-LLM pipeline to identify and selectively erase "forget-relevant" segments from CoT traces, achieving compliance without degrading overall reasoning capability.

**Key Contributions:**
- Identifies and formalizes a **new compliance gap** in deploying reasoning models: CoT traces leak sensitive information even after standard unlearning.
- Proposes a retrieval-augmented pipeline to detect and surgically remove forget-relevant reasoning segments.
- Demonstrates that selective CoT forgetting can meet regulatory requirements (e.g., GDPR right-to-be-forgotten) while preserving model utility.

## Quick Picks

### Must-Read #1: **GrandCode** (2604.02721)
This is a genuine milestone paper. AI surpassing all human grandmasters in live, unmodified Codeforces competitions is a historically significant result — analogous to AlphaGo in Go or Stockfish in Chess, but for open-ended programming under real competition pressure. The Agentic GRPO algorithm is also a novel and reusable contribution for the broader multi-agent RL community.

### Must-Read #2: **VLMs Need Words** (2604.02486)
If you work with vision-language models in any applied context, this paper is essential reading. The "semantic anchor" finding challenges the common assumption that VLMs are capable visual reasoners — and has direct implications for high-stakes domains like medical imaging, fine-grained product recognition, and face verification. Understanding this limitation now will save significant debugging time later.
